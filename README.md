# Professional Portfolio & Digital Resume

Welcome to the repository for my professional portfolio website. This project serves as a comprehensive digital showcase designed to present my technical capabilities, career progression, and quality assurance achievements. 

The primary purpose of this website is to provide prospective employers and collaborators with an interactive overview of my work. The application is logically structured into three main components: a digital Resume, an API Automation Showcase, and a UI Automation Showcase.

---

## Feature Breakdown

### Resume

This section acts as my complete digital resume, offering a detailed narrative of my professional journey. Visitors exploring this section will find:

* **Professional Experience:** A comprehensive timeline of my previous roles, highlighting my key responsibilities and contributions to various software **programmes**.
* **Core Competencies:** A structured list of my technical skills, focusing on software quality assurance, test automation, and front-end development.
* **Education:** Details regarding my academic background, degrees achieved, and relevant industry certifications.

### API Automation Showcases

A prominent, dedicated space to highlight my expertise in validating and securing backend services. This section details:

* **Frameworks & Strategies:** In-depth overviews of the architectures and tools I utilise to **analyse** complex system **behaviour**.
* **Project Breakdowns:** Clear demonstrations of how I design, implement, and **optimise** API testing suites for enterprise-scale applications.
* **Key Metrics:** Tangible achievements and data points, such as exact percentages by which I have reduced regression testing time.

### UI Automation Showcases

This component focuses on my ability to ensure robust, flawless user experiences across the frontend. It features:

* **Frontend Testing Projects:** Real-world examples of end-to-end (E2E) testing implementations and component-level validation.
* **Tech Stack & Tooling:** Insights into the specific libraries (e.g., Cypress, Playwright) I use to **customise** UI automation pipelines.
* **QA Achievements:** Specific metrics detailing how these automated UI tests have caught critical defects and elevated overall product reliability.
